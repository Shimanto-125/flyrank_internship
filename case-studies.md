# Case Study 1
## Applied Search Intelligence with the FlyRank ML Internship

### The Problem

Search engines generate large amounts of keyword and ranking data, making it difficult to manually identify the best opportunities for improving website visibility. Manual analysis is slow, inconsistent, and difficult to reproduce.

The objective of this internship project is to build a structured, repeatable workflow that transforms search data into useful insights and ranked content recommendations.

---

### What I Did and Why

I began by exploring the FlyRank starter dataset to understand the available features and identify what information could be used for search intelligence.

Instead of immediately building a machine learning model, I first defined the research question, clarified the business objective, and determined what type of machine learning task best matched the problem.

My work included:

- Exploring the provided search dataset.
- Identifying useful ranking-related features.
- Studying how search performance can be measured.
- Comparing different machine learning task formulations.
- Designing a repeatable analysis workflow.
- Documenting assumptions and design decisions.
- Preparing notebooks and documentation that can be reproduced by others.

One important decision was focusing on understanding the problem before selecting an algorithm. This helped ensure that every modelling choice could be explained and justified rather than selected simply because it was popular.

---

### Outcome

The project established a clear workflow for analyzing search intelligence data and preparing ranked content recommendations.

Although the internship project is still progressing, the work completed so far provides:

- A reproducible research workflow.
- A structured process for analyzing search datasets.
- Clear documentation of decisions.
- A strong foundation for future machine learning experiments.

The biggest lesson from this project has been that understanding the business problem is just as important as building the machine learning model.

# Case Study 2
## Heart Disease Prediction Using Neural Networks

### The Problem

Healthcare datasets often contain missing values, inconsistent formats, and mixed numerical and categorical variables. These issues make machine learning models difficult to train effectively.

The goal of this project was to build a neural network capable of predicting heart disease while understanding how data preprocessing affects model performance.

---

### What I Did and Why

I cleaned the dataset by handling missing values, removing unusable features, encoding categorical variables, and standardizing numerical data before training the model.

The project included:

- Data cleaning.
- Missing value handling.
- Feature selection.
- Label encoding.
- Feature scaling.
- Neural network implementation using PyTorch.
- Model training.
- Performance evaluation.
- Error analysis.

During development I encountered several issues, including missing values causing training failures and unstable predictions. Rather than ignoring these errors, I investigated their causes, corrected the preprocessing pipeline, retrained the model, and compared the results.

This project reinforced the importance of careful data preparation before model development.

---

### Outcome

The project successfully demonstrated the complete supervised learning workflow from raw data preprocessing to model evaluation.

More importantly, it showed that model quality depends heavily on data quality, preprocessing decisions, and continuous debugging rather than simply choosing a more complex algorithm.
